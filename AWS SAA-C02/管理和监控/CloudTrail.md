AWS CloudTrail 是 AWS 审计与安全策略的核心，负责记录账户内由主体（用户或角色）发起的所有 API 调用和非 API 操作（如控制台登录）

## 详尽的操作追踪
- 审计内容 : 每一条日志都记录了操作时间、身份详情（如 IAM 用户名或 ARN）、API 名称、受影响资源以及来源 IP 地址
- 事件分类: 它区分“管理事件”（如创建 VPC 的控制层面操作）和“数据事件”（如 S3 对象访问或 Lambda 执行等高频率操作）
- 多区域整合: 虽然事件历史默认按区域存储，但用户可以创建跨区域的“跟踪”(Trail)，将全球所有区域的操作记录集中交付到指定的 S3 存储桶中

## 管理与监控的协同发
- 与[[AWS Config]]的互补: CloudTrail 专注于记录“谁在何时做了什么”，而 AWS Config 则关注这些动作导致的资源“配置状态”变化，两者结合可提供完整的变更溯源
- 集成[[CloudWatch]] : CloudTrail 可以将日志流式传输到 CloudWatch Logs，使用户能够实时搜索特定的 API 事件，或通过指标筛选器触发警报

## 安全和合规性
- 完整性验证: CloudTrail 提供日志文件完整性验证功能，利用加密哈希摘要确保日志在交付到 S3 后未被恶意篡改或删除
- 威胁侦测: 它是 Amazon GuardDuty 的关键数据源之一。GuardDuty 通过分析 CloudTrail 管理日志来识别凭证被盗、可疑的 API 调用或攻击者的侦察行为

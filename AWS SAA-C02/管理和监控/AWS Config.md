在 AWS 的“身份、管理与监控”体系中，**AWS Config** 扮演着“资源时间机器”的角色，专注于跟踪和审计资源配置的变化

## 状态追踪和时间机器功能
- 记录配置状态: AWS Config 会发现现有资源，记录其详细设置，并持续监控后续的任何变动
- 配置项和历史: 它会为每个资源生成“配置项”(Configuration Item)，包含特定时间点的设置及与其他资源的关系。这些记录汇编成“配置历史”，让您能追溯资源在过去任何时刻的真实状态
- 关系分析: 它能展示资源间的关联，例如修改一个安全组会如何影响所有关联的 EC2 实例，这对于变更管理至关重要

## 合规性和管理规则
- 基准定义: 通过“AWS Config规则”， 可以定义理想的配置基准（例如“确保所有 EBS 卷都已加密”或“CloudTrail 必须开启”
- 自动合规评估： 当资源配置偏离基准时，系统会自动将其标记为“非合规”并通过 SNS 发送通知，帮助您满足审计和监管要求

## 在监控生态中的互补作用
- 与[[CloudTrail]]协作 ： CloudTrail记录谁在什么时候做了什么（事件日志），而AWS Config记录这些动作产生的最终状态变化（配置状态）， 两者结合可以提供完整的操作与状态溯源
- 与[[CloudWatch]]协作： CloudWatch关注资源运行的好不好（性能指标），而AWS Config关注资源是什么样子的（配置细节）

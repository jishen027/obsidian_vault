**Amazon Kinesis** 是一组专门用于收集、处理、存储和交付**实时流数据**的服务集合。它能够每秒处理来自数千个源的千兆字节数据，非常适合日志、遥测、音频和视频流

## Kinesis Data Streams (数据流)
这是最基础的管道服务，用于收集并暂存数据，直到消费者准备好处理它
- **工作模式：** 采用“生产者-消费者”模型
- **核心组件：** 数据通过**分片 (Shards)** 进行管理，每个分片有固定的吞吐量限制（如 1MB/s 写入，2MB/s 读取）
- **特性：** 支持多个消费者同时读取同一份数据（扇出/Fan-out）
- **保留期：** 默认保留 24 小时，最多可延长至 **7 天**

## Kinesis Data Firehose (交付流)
它是最简单的交付方案，专注于将数据加载到特定的 AWS 目的地
- **主要功能：** 能够自动缩放，并在交付前通过 **[[Lambda]]** 函数转换数据格式（如将 JSON 转为 Parquet）
- **目的地：** 常见的包括 [[S3 - Object Storage]] S3、[[数据仓库服务 - Redshift]]、Elasticsearch 或 Splunk
- **区别：** 它没有“消费者”订阅模式，而是直接将数据投递到目的地

## Kinesis Video Streams (视频流)
专门用于传输和存储时间索引的视频数据
- **应用场景：** 安全摄像头、智能手机视频、机器视觉等
- **保留期：** 同样默认 24 小时，最长 7 天

## **核心对比总结**

| 特性       | Data Streams | Data Firehose  |
| :------- | :----------- | :------------- |
| **数据处理** | 自定义消费者程序处理   | 自动投递到 AWS 目标   |
| **延迟**   | 亚秒级          | 60 秒或以上（取决于缓冲） |
| **可扩展性** | 需要手动管理分片数量   | 完全托管，自动缩放      |

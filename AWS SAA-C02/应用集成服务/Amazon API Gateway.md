# Amazon API Gateway

> **Amazon API Gateway** 是一种完全托管的服务，用于创建、发布、维护、监控和保护 REST、HTTP 和 WebSocket API。它是构建 API 驱动的微服务架构和 Serverless 应用的核心服务。
>
> 相关文档：[[EC2|EC2]] | [[Lambda|Lambda]] | [[CloudWatch]]

---

## 核心概念

### API Gateway 定位

| 服务                                                | 类型            | 场景           | 特点         |
| ------------------------------------------------- | ------------- | ------------ | ---------- |
| **API Gateway**                                   | API 管理        | API 创建、发布、监控 | 流量入口、认证、限流 |
| [[Lambda\|计算服务 - Lambda]] | Serverless 计算 | 事件驱动函数执行     | 无需管理服务器    |
| [[EC2\|计算服务 - EC2]]       | 虚拟机           | 传统应用托管       | 完全控制服务器    |

### 支持的 API 类型

| API 类型            | 协议               | 适用场景             |
| ----------------- | ---------------- | ---------------- |
| **REST API**      | RESTful (HTTP)   | 最常用，向后兼容         |
| **HTTP API**      | HTTP/1.1, HTTP/2 | 轻量级、低成本、低延迟      |
| **WebSocket API** | WebSocket        | 实时双向通信（聊天、游戏）    |
| **v2 API**        | REST/HTTP 升级版    | 新增功能（自定义认证、路由变量） |

## 架构组件

### API 组成结构

```
                    ┌─────────────────────────┐
                    │      API Gateway         │
                    │  ┌───────────────────┐   │
        Client ───→ │  │   Stage/Deploy    │   │──→ Backend
                    │  │  (Prod / Dev)     │   │
                    │  └───────────────────┘   │
                    │  ┌───────────────────┐   │
                    │  │  Resources & Methods│  │
                    │  └───────────────────┘   │
                    └─────────────────────────┘
```

| 组件 | 描述 |
|------|------|
| **API** | 一组资源和方法的集合 |
| **Resource** | API 中的端点路径（如 `/users`、`/products/{id}`） |
| **Method** | 资源上的操作（GET、POST、PUT、DELETE） |
| **Stage** | API 的部署版本（如 dev、staging、prod） |
| **Deployment** | 将 API 配置部署到 Stage |

### 后端集成类型

| 集成类型 | 描述 | 适用场景 |
|---------|------|---------|
| **Lambda 集成** | 调用 Lambda 函数 | Serverless 架构（最常用） |
| **HTTP 集成** | 调用 HTTP 端点 | 微服务、外部 API |
| **Mock 集成** | 返回模拟响应 | 开发和测试 |
| **AWS 服务集成** | 调用其他 AWS 服务 | S3、DynamoDB 等 |
| **VPC Link** | 调用 VPC 内资源 | 私有后端服务 |

## 认证和授权

### 认证方式

| 方式 | 说明 | 适用场景 |
|------|------|---------|
| **IAM 认证** | 使用 AWS IAM 凭证 | AWS 服务间调用 |
| **API Key** | 简单的密钥验证 | 免费 API、限流 |
| **Cognito User Pools** | AWS 用户身份池 | 用户认证和授权 |
| **Lambda 授权** | 自定义 Lambda 函数验证 | 复杂认证逻辑 |
| **OIDC/JWT** | OpenID Connect 令牌 | 现代应用、微服务 |

### 授权类型对比

| 授权类型 | 控制层级 | 灵活性 | 性能 |
|---------|---------|--------|------|
| **IAM** | API 级别 | 低 | 高（原生支持） |
| **API Key** | API 级别 | 低 | 高 |
| **Cognito** | 用户级别 | 中 | 中 |
| **Lambda (自定义)** | 请求级别 | 高 | 低（额外 Lambda 调用） |
| **JWT** | 令牌级别 | 高 | 高（API Gateway 原生验证） |

## 性能优化

### 缓存机制

| 特性 | 说明 |
|------|------|
| **缓存位置** | API Gateway 边缘节点 |
| **缓存 TTL** | 0-3600 秒（默认 300 秒） |
| **缓存密钥** | 基于请求参数、头、方法 |
| **适用场景** | 读取频繁、写入少端的 API |
| **成本** | 按缓存容量计费 |

### Lambda 集成优化

| 优化项 | 说明 |
|--------|------|
| **Lambda 冷启动** | 使用 Provisioned Concurrency 预热 |
| **连接复用** | HTTP API 使用 HTTP/2 连接复用 |
| **Payload 压缩** | 启用压缩减少传输时间 |
| **超时设置** | 合理设置超时时间（默认 29 秒） |

### API 类型性能对比

| 特性 | REST API | HTTP API | WebSocket API |
|------|---------|---------|--------------|
| 延迟 | 中 | 低（500-700ms） | 低 |
| 成本 | 高 | 低（40-70% 节省） | 中 |
| 协议 | HTTP/1.1 | HTTP/1.1, HTTP/2 | WebSocket |
| 连接持久化 | 否 | 否 | 是 |

## 安全和监控

### 安全防护

| 安全特性 | 说明 |
|---------|------|
| **DDoS 保护** | 与 AWS Shield 集成 |
| **WAF 集成** | 与 AWS WAF 集成，防止 SQL 注入、XSS 等 |
| **TLS 加密** | 传输中数据加密 |
| **VPC Link** | 私有后端访问，不暴露公网 |
| **速率限制** | 限制请求频率，防止滥用 |
| **配额管理** | 限制请求总量 |

### 速率限制和配额

| 特性 | 说明 |
|------|------|
| **速率限制 (Rate Limiting)** | 每秒请求数限制 |
| **突发限制 (Burst Limit)** | 瞬时最大请求数 |
| **配额 (Quotas)** | 时间窗口内总请求数 |
| **计数头** | 响应中包含剩余请求计数 |

### 监控和日志

| 服务 | 功能 |
|------|------|
| **CloudWatch** | 监控延迟、错误数、调用次数 |
| **访问日志** | 记录每个请求的详细信息 |
| **追踪日志** | 记录 Lambda 执行细节 |
| **CloudTrail** | 审计 API Gateway API 调用 |
| **X-Ray** | 分布式追踪 |

## 生命周期管理

### API 开发流程

```
创建 API → 定义资源和方法 → 配置集成 → 测试 → 部署到 Stage → 监控和优化
```

### 版本管理

| 阶段 | 说明 |
|------|------|
| **Development** | 开发环境测试 |
| **Staging** | 预发布环境验证 |
| **Production** | 生产环境 |
| **Canary** | 灰度发布（逐步切换流量） |

### 部署管理

- **API 版本**：支持版本化（如 `/v1/users`、`/v2/users`）
- **Stage 变量**：不同环境使用不同配置
- **部署快照**：每次部署创建快照，可回滚

## 考试重点总结

### SAA-C02 高频考点

1. **API 类型选择**：HTTP API（低成本、低延迟）、REST API（功能全面）、WebSocket（实时通信）
2. **缓存**：减少后端负载，基于请求参数
3. **认证方式**：IAM（服务间）、API Key（免费 API）、Cognito（用户认证）、Lambda 授权（自定义）
4. **JWT/OIDC**：API Gateway 原生支持，无需 Lambda
5. **VPC Link**：访问 VPC 内私有资源
6. **速率限制**：防止 API 滥用
7. **WAF 集成**：防止 Web 攻击
8. **Stage 和部署**：环境隔离和版本管理
9. **Lambda 集成**：Serverless 架构核心
10. **CloudWatch 监控**：延迟、错误数、调用次数

### 场景题解题思路

```
场景分析 → 选择 API 类型和配置
├── "需要最低成本和延迟" → HTTP API
├── "需要实时双向通信" → WebSocket API
├── "需要完整功能" → REST API
├── "需要保护后端 VPC 资源" → VPC Link
├── "需要 JWT 认证" → 原生 JWT 认证
└── "需要防止 DDoS/SQL 注入" → WAF 集成

**AWS CloudFormation** 是一种“基础设施即代码 (IaC)”服务，允许您使用 JSON 或 YAML 格式的文本文件来定义和自动化部署整个 AWS 基础架构

## 核心组件
- **模板 (Templates)：** 描述您所需资源及其配置的声明式文本文件。它是您架构的蓝图
- **堆栈 (Stacks)：** 通过模板创建的一组 AWS 资源的单一管理单元。您可以作为一个整体来创建、更新或删除堆栈
- **逻辑 ID vs 物理 ID：** 模板中定义的资源名称称为**逻辑 ID**，而 AWS 实际创建出的资源 ID（如 `vpc-0380...`）称为**物理 ID**

## 关键功能
- **参数 (Parameters)：** 允许在创建堆栈时动态输入值（如选择特定的 AMI 或密钥对名称），从而避免将固定值硬编码在模板中
- **嵌套堆栈 (Nested Stacks)：** 允许一个堆栈创建另一个堆栈。这有助于实现模块化，例如网络团队维护一个底层的网络堆栈，而开发团队在之上构建应用堆栈
- **变更集 (Change Sets)：** 在正式更新堆栈前，通过变更集预览 CloudFormation 将如何修改资源，以防止意外的停机或资源替换

## 安全与治理
- **堆栈策略 (Stack Policies)：** 这是一份 JSON 文档，用于定义哪些资源可以在更新期间被修改。它可以防止核心资源（如数据库）在堆栈更新时被意外替换或删除
- **漂移检测：** 虽然资料中未详尽提及，但 CloudFormation 支持检测实际资源状态与模板定义之间的差异

## 成本与优势
- **成本：** CloudFormation 服务本身是**免费**的，您只需为您通过它创建出来的 AWS 资源（如 EC2 实例）
- **优势：** 实现环境的一致性（生产和开发环境完全相同）、自动化快速部署、以及将基础架构作为版本控制的代码进行管理

1. 子网： 资源的逻辑容器， 子网是从[[虚拟私有云 - VPC]]的CIDR 地址块中划分出来的较小的IP范围
	1. 可用区绑定[[虚拟私有云 - VPC#^6bde30]]：每个子网必须位于一个可用区， 通过在不同可用区的子网中分布实例， 可以实现高可用性
	2. IP地址的限制： AWS 会保留每个子网地址范围的前4个和最后1个IP地址用于路由和DNS 服务
	3. 不可移动性： EC2 实例在启动时必须关联子网，且无法在子网间移动， 只能通过AMI 重建
2. 路由表- Route table：流量的指挥官，路由表包含一系列的路由规则，用于判断网络流量的去向
	1. 隐式路由器-Implicit Router : VPC 通过软件抽象出路由器根据路由表执行目标地址匹配
	2. 本地路由-Local Router: 每个路由都包含默认的本地路由，允许同一VPC 内不同子网之间的实例进行通信
	3. 匹配规则： 遵循最长前缀匹配，流量会优先匹配地址描述最精确的路由规则
3. 公有子网和私有子网
	1. 公有子网[[虚拟私有云 - VPC#^f9405b]] ^8377ca
	2. 私有子网[[虚拟私有云 - VPC#^96eac8]]
4. 关联和管理
	1. 默认关联： 每个VPC 都有一个主路由表-Main Route Table, 如果新建子网时未指定路由表，会自动关联上主路由表
	2. 一对多关系： 一个子网对一个路由表，一个路由表对多个子网

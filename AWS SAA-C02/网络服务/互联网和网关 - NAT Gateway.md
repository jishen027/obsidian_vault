IGW 和 NAT 设备时定义子网性质和控制流量进出VPC 的关键组件

## 互联网网关 - IGW
1. 核心功能： IGW为VPC提供连接互联网的的能力，允许实例获取共有IP 并接受来自互联网的请求，同时也允许实例主动访问互联网
2. 公有子网的定义[[子网和路由表 - Subnet & Route table#^8377ca]] ： 子网中包含一个指向IGW ID 的默认路由
3. 管理特性： 没有物理IP，每个VPC 只能关联一个IGW

## NAT 设备 - Network Address Translation
1. 核心功能： NAT 设备允许私有子网中的实例访问互联网，同时防止互联网的外部主机主动发动连接到实例
2. NAT 网关 - NAT Gateway : AWS 提供的托管服务，高可用和自动扩展，必须部署在公有子网中并且分配一个弹性IP
3. NAT 实例 - NAT Instance : 运行特定AMI的实例，需要手动管理并且禁用网络结构上的“源/目的地检查”

## 网络架构中的写作
1. 路由逻辑： 私有子网中的实例通过路由表将默认流量指向NAT设备，而NAT 设备随后通过公有子网的路由表将流量转发至IGW，最终到达互联网。
2. 安全性： 这种架构设计实现了计算机资源的隔离，确保了服务器在保持互联网更新的同时不直接暴露在公共网络中

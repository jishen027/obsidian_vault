网络接口是连接实例与网络的桥梁。根据不同的性能需求和应用场景，主要分为以下几种类型

## ENI
这是最基础的逻辑联网组件，代表一个虚拟网卡
- **核心组成：** 每个 ENI 包含一个主要私有 IPv4 地址、MAC 地址、安全组关联以及源/目的地检查标志
- **灵活性：** 你可以创建 ENI 并将其动态地从一个实例分离并重新附加到另一个实例，从而实现 IP 地址或网络配置的快速迁移
- **限制：** ENI 必须与其附加的实例处于同一个可用区（AZ）和子网内

## Enhanced Networking
为了提供更高的每秒数据包处理能力（PPS）和更低的延迟，AWS 提供了增强型网络技术，主要通过单根 I/O 虚拟化 (SR-IOV) 实现
- **ENA (Elastic Network Adapter)：** 这是目前的主流标准，支持高达 **100 Gbps** 的网络带宽
- **Intel 82599 VF 接口：** 较旧的技术，最高支持 **10 Gbps**，仅用于部分老旧实例类型

## EFA
这是专为**高性能计算 (HPC)** 和机器学习设计的特殊 ENI
- **技术特性：** 允许应用程序通过 Libfabric API 实现“操作系统旁路”(OS Bypass)，直接跳过标准 TCP/IP 堆栈进行通信，极大地降低了延迟
- **适用场景：** 紧耦合的集群计算，通常配合 MPI（消息传递接口）使用
- **部署要求：** 所有使用 EFA 的实例必须位于**同一个子网**内，且 EFA 流量不可路由

## Direct Connect 虚拟接口 (VIF)
在通过专用物理线路连接本地机房与 AWS 时使用
- **私有 VIF：** 用于连接单个 VPC 中的私有资源（如 EC2、RDS）
- **公有 VIF：** **公有 VIF：** 允许通过专用线路访问具有公网端点的 AWS 服务（如 S3、DynamoDB）
- **中转 VIF：** 专门用于连接 **Transit Gateway**，从而实现单条线路互联成百上千个 VPC

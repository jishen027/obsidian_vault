**安全组（Security Group）** 是 AWS 中最基础且最重要的安全组件之一，它的核心身份是**虚拟防火墙**，专门用于控制单个或多个 EC2 实例（以及其他资源，如 RDS、ALB）的入站和出站流量

## 核心工作原理：有状态（Stateful）

这是安全组最显著的特征。**“有状态”****自动被允许**进入实例，即使入站规则里没有明确放行该流量 。同理，如果入站请求被允许，其响应的出站流量也会自动放行 。

## 默认规则：入站全关，出站全开

- **入站（Inbound）：** 新创建的安全组默认**拒绝所有**入站流量 。你必须手动添加“白名单”规则来允许特定的流量（如 SSH 的 22 端口或 HTTP 的 80 端口）。
- **出站（Outbound）：** 默认**允许所有**流量离开实例（目的地通常设为 `0.0.0.0/0`）

## 规则的构成要素
每一条规则都需要定义以下三个核心参数：
- **协议（Protocol）：** 如 TCP、UDP、ICMP 等
- **端口范围（Port Range）：** 如 80 (HTTP)、443 (HTTPS) 或 22 (SSH)
- **源/目的地（Source/Destination）：** 可以是具体的 IP 地址段（CIDR），也可以是**另一个安全组 ID**
	- _小技巧：_ 将另一个安全组设为“源”，可以轻松实现“只允许来自特定 Web 服务器组的请求访问数据库层”
## 安全组 vs. 网络 ACL (NACL)

在考试和实际架构中，经常需要区分这两者： 

| 特性       | 安全组 (Security Group)                 | 网络 ACL (NACL)  |
| -------- | ------------------------------------ | -------------- |
| **作用范围** | 实例/网卡 ([[Network Interface#ENI]]) 级别 | 子网 (Subnet) 级别 |
| **状态属性** | 有状态（Stateful）                        | 无状态（Stateless） |
| **规则类型** | 仅支持“允许”规则（白名单）                       | 支持“允许”和“拒绝”规则  |
| **生效顺序** | 评估所有规则后决定是否放行                        | 按规则编号从小到大顺序匹配  |

# S3 - Simple Storage Service

> **Amazon S3 (Simple Storage Service)** 是 AWS 提供的分布式对象存储服务，为任何规模的数据提供高持久性、高可用性的存储。是 AWS 架构中最基础和最常用的服务之一。
>
> 相关文档：[[存储服务 - Storage Service]] | [[S3 Glacier Archive]] | [[块储存 - EBS]]

---

## 核心概念

### 对象存储 vs 块存储

| 特性 | S3 (对象存储) | [[块储存 - EBS]] (块存储) |
|------|--------------|-------------------------|
| 数据结构 | 扁平命名空间，对象由 Key 标识 | 将数据划分为块，由文件系统管理 |
| 访问方式 | REST API (HTTP/HTTPS) | 块设备挂载（如 `/dev/sdf`） |
| 适用场景 | 静态资源、备份归档、大数据分析 | 操作系统盘、数据库、企业级应用 |
| 一致性模型 | 强一致性（2020 年后） | 强一致性 |

### 基本组成

| 组件 | 描述 |
|------|------|
| **Bucket (存储桶)** | S3 的全局唯一命名容器，类似文件夹根目录 |
| **Object (对象)** | S3 中的基本存储单元，包含数据、元数据和 Key |
| **Key (键)** | 对象的唯一标识符，支持文件夹结构（`prefix/prefix/key`） |
| **Version (版本)** | 启用版本控制后，每个对象有多个版本 |

## 组织架构和规模

### Bucket 命名规则

- **全局唯一**：Bucket 名称在 AWS 全局范围内唯一
- 长度：3-63 个字符
- 只能包含小写字母、数字、点和连字符
- 不能是 IP 地址格式

### 容量限制

| 限制项 | 值 |
|--------|-----|
| 单个对象大小 | **5 GB - 5 TB**（大于 100 MB 建议使用分块上传） |
| Bucket 容量 | **无限** |
| 每个 Bucket 的对象数 | **无限** |
| 对象 Key 最大长度 | 1,024 个字符 |

### 分块上传 (Multipart Upload)

- **分块大小**：100 KB - 5 GB（推荐 50-100 MB）
- **最大块数**：10,000 个块
- **优势**：
  - 大文件上传的可靠性和性能
  - 中断后可从断点恢复
  - 并行上传提高效率

## 存储类别

### S3 存储类别对比（考试重点）

| 存储类别 | 持久性 | 可用性 | 最小存储期限 | 检索时间 | 适用场景 |
|---------|--------|--------|------------|---------|---------|
| **S3 Standard** | 11个9 | 99.99% | 无 | 即时 | 频繁访问的热数据 |
| **S3 Intelligent-Tiering** | 11个9 | 99.99% | 无 | 即时 | 访问模式未知或变化的数据 |
| **S3 Standard-IA** | 11个9 | 99.9% | 30 天 | 即时 | 不频繁访问但需要快速检索 |
| **S3 One Zone-IA** | 99.5% | 99.5% | 30 天 | 即时 | 可重新创建的非关键数据 |
| **S3 Glacier Flexible** | 11个9 | 99.99% | 90 天 | 分钟-小时 | 归档数据，定期访问 |
| **S3 Glacier Deep Archive** | 11个9 | 99.99% | 180 天 | 12-48 小时 | 长期合规归档 |

### 存储类别选择指南（考试场景题）

```
需要即时访问？
├── 是 → 频繁访问？
│         ├── 是 → S3 Standard
│         └── 否 → 访问模式可预测？
│                   ├── 是 → S3 Standard-IA
│                   └── 否 → S3 Intelligent-Tiering
└── 否 → 可接受延迟？
          ├── 分钟-小时 → S3 Glacier Flexible Retrieval
          └── 12-48 小时 → S3 Glacier Deep Archive
```

### 存储类别成本对比

| 存储类别 | 存储成本 | 检索成本 | 适合数据比例 |
|---------|---------|---------|------------|
| S3 Standard | 最高 | 无 | 100% 热数据 |
| S3 Standard-IA | 低 50% | 按检索量计费 | 冷数据 |
| S3 One Zone-IA | 低 60% | 按检索量计费 | 可重新创建的数据 |
| S3 Glacier | 低 95% | 按检索时间和时长计费 | 归档数据 |
| S3 Glacier Deep Archive | 低 96% | 按检索时间和时长计费 | 长期归档 |

## 持久性和可用性

### 数据保护机制

| 特性 | 说明 |
|------|------|
| **持久性** | **99.999999999% (11个9)**，设计为每年丢失 0.005% 以下的数据 |
| **可用性** | S3 Standard：99.99%（SLA） |
| **数据复制** | 自动在多个可用区 (AZ) 冗余存储（One Zone-IA 除外） |

### 版本控制

- 启用后，对对象的每次编辑都会生成新版本
- 可以恢复意外删除或覆盖的对象
- **不能暂停**，只能启用或禁用
- 适用于防止人为错误和合规要求

### 生命周期管理

- 自动将对象转换到更经济的存储类别
- 可配置规则：
  - 基于时间的转移（如 30 天后转移到 IA，90 天后转移到 Glacier）
  - 基于前缀或标签的转移
  - 过期后自动删除

#### 典型生命周期配置（考试常见场景）

```
创建 → S3 Standard (热数据)
        ↓ (30天后)
     S3 Standard-IA
        ↓ (90天后)
     S3 Glacier
        ↓ (180天后)
     S3 Glacier Deep Archive
        ↓ (7年后)
     自动删除
```

## 安全性

### 访问控制模型

| 机制 | 描述 |
|------|------|
| **IAM 策略** | 基于身份的权限策略，控制谁可以访问 S3 |
| **Bucket Policy** | 附加到 Bucket 的资源型策略 |
| **ACL (访问控制列表)** | 细粒度控制 Bucket 和对象的访问权限 |
| **S3 Access Points** | 为 S3 提供自定义入口，简化数据访问管理 |
| **Object Lambda** | 在访问对象时实时转换数据 |

### 加密

| 加密类型 | 服务 | 说明 |
|---------|------|------|
| **SSE-S3** | S3 管理密钥 | 默认加密，最简单 |
| **SSE-KMS** | AWS KMS | 密钥管理更精细，审计更强 |
| **SSE-C** | 客户管理密钥 | 客户负责密钥管理 |
| **客户端加密** | 客户端 | 上传前加密 |

### 访问日志和监控

- **S3 Access Logging**：记录访问 Bucket 的请求
- **S3 Inventory**：生成对象清单，用于合规和自动化
- **CloudTrail**：审计 S3 API 调用
- **CloudWatch Metrics**：监控存储量和请求数

## 网络访问

### S3 存储桶 URL 格式

| 类型 | 格式 | 示例 |
|------|------|------|
| **虚拟托管式** | `https://bucket-name.s3.region.amazonaws.com/key` | `https://mybucket.s3.us-east-1.amazonaws.com/photo.jpg` |
| **路径式** | `https://s3.region.amazonaws.com/bucket-name/key` | `https://s3.us-east-1.amazonaws.com/mybucket/photo.jpg` |
| **虚拟主机式 (DNS)** | `https://bucket-name.s3.amazonaws.com` | 用于 us-east-1 区域 |

### VPC 终端节点 (VPC Endpoint)

- **类型**：接口终端节点 (Interface Endpoint)
- **功能**：让 VPC 内的资源通过私有网络访问 S3，不经过公网
- **优势**：
  - 安全性：流量不暴露在公共互联网
  - 成本：无需 NAT 网关费用
  - 性能：更低延迟，更稳定
- **配合 Bucket Policy**：限制只有通过终端节点才能访问 Bucket

### S3 访问方式对比

| 方式 | 网络路径 | 费用 | 安全性 |
|------|---------|------|--------|
| 互联网 + IGW | 通过公网 | 数据传输费 | 较低 |
| VPC Endpoint | 私有网络 | 无数据传输费 | 高 |
| CloudFront + Origin Access | CDN 缓存 | 低延迟 | 高 |
| Storage Gateway | 混合云 | 额外费用 | 高 |

## 高级功能

### CloudFront 集成

- S3 可作为 CloudFront 的源站
- **Origin Access Control (OAC)**：保护私有 Bucket，只允许 CloudFront 访问
- CDN 缓存静态内容，降低延迟和 S3 请求成本

### 签名 URL (Signed URL)

- **用途**：临时授权访问私有对象
- **机制**：URL 包含签名认证字符串，在有效期内有效
- **特点**：
  - 过期后自动失效
  - 可指定访问权限（读/写）
  - 使用签名密钥完成身份验证
- **适用场景**：用户临时下载/上传私有文件

### 事件通知

- S3 可触发 **Lambda 函数**（如图片缩略图生成）
- 可发送消息到 **SQS** 或 **SNS**
- 适用于自动化数据处理流水线

### S3 Replication

| 类型 | 说明 |
|------|------|
| **CRR (跨区复制)** | 跨区域复制，满足合规要求 |
| **SRR (同区复制)** | 同区域内跨 Bucket 复制，自动版本复制 |

## 考试重点总结

### SAA-C02 高频考点

1. **存储类别选择**：根据访问频率、延迟要求、成本选择正确类别
2. **生命周期策略**：自动转换存储类别降低成本
3. **版本控制**：防止意外删除和覆盖
4. **加密**：SSE-S3、SSE-KMS、SSE-C 的区别和选择
5. **VPC Endpoint**：私有访问 S3，避免数据经过公网
6. **CloudFront + S3**：Origin Access 保护私有 Bucket
7. **事件驱动架构**：S3 事件触发 Lambda
8. **一致性模型**：S3 提供强一致性（读后写一致性）
9. **存储类别成本排序**：Standard > IA > One Zone-IA > Glacier > Deep Archive
10. **分块上传**：大于 100 MB 使用分块上传

### 常见场景题解题思路

```
场景题 → 提取关键词 → 匹配存储类别
├── "频繁访问" → S3 Standard
├── "不频繁但需即时" → S3 Standard-IA
├── "访问模式未知" → Intelligent-Tiering
├── "可重新创建" → One Zone-IA
├── "归档，分钟级检索" → Glacier Flexible
└── "归档，12-48小时" → Glacier Deep Archive
```

## 最佳实践

1. **选择正确的存储类别**：根据访问模式选择，使用 Intelligent-Tiering 自动化
2. **启用版本控制**：防止意外删除和覆盖
3. **配置生命周期策略**：自动转移冷数据降低成本
4. **启用加密**：默认使用 SSE-S3 或 SSE-KMS
5. **使用 VPC Endpoint**：VPC 内访问 S3 时保护数据安全
6. **CloudFront 缓存**：静态内容使用 CloudFront 加速
7. **设置 Bucket 日志**：审计和监控访问
8. **使用标签**：组织和管理对象

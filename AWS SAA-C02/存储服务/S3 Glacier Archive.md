# S3 Glacier - 归档存储服务

> **Amazon S3 Glacier** 是 AWS 提供的超低成本对象存储服务，专为长期归档和灾难恢复设计。提供多种检索选项以平衡成本和访问速度需求。
>
> 相关文档：[[S3 - Object Storage]] | [[存储服务 - Storage Service]]

---

## 核心概念

### 存储架构

| 概念 | 描述 |
|------|------|
| **Archive (档案)** | S3 Glacier 中的基本存储单元，单个档案最大可达 **40 TB** |
| **Vault (保险库)** | 存储档案的容器，类似 S3 的 Bucket |
| **Vault Lock** | 通过策略锁定保险库，防止档案被修改或删除（WORM 模型） |

### 与 S3 的关系

- S3 Glacier 是 **S3 的存储类别**，而非独立服务
- 通过 S3 API 统一管理，无需切换服务
- 支持 **S3 生命周期策略**自动将对象转移到 Glacier 存储
- 相关文档：[[S3 - Object Storage]]

## 存储类别

### S3 Glacier 存储类别对比

| 存储类别 | 最小存储期限 | 检索时间 | 适用场景 |
|---------|-------------|---------|---------|
| **Glacier Instant Retrieval** | 无 | 毫秒至秒级 | 需要即时访问的归档数据 |
| **Glacier Flexible Retrieval** (原 Glacier) | 90 天 | 分钟到小时级 | 标准的归档存储（最常用） |
| **Glacier Deep Archive** | 180 天 | 12-48 小时 | 长期合规和极低成本归档 |

### 存储成本对比

| 存储类别 | 每 GB 月成本 (约) | 对比 Standard |
|---------|------------------|--------------|
| S3 Standard | $0.023 | 基准 |
| S3 IA | $0.0125 | 低 46% |
| **Glacier Flexible Retrieval** | **$0.00099** | **低 96%** |
| **Glacier Deep Archive** | **$0.00099** | **低 96%** |

> Glacier Deep Archive 是 AWS 最低成本的存储服务，比 S3 Standard 便宜约 96%。

## 检索选项

### 检索速度 vs 成本

| 检索类型 | 完成时间 | 费用 | 适用场景 |
|---------|---------|------|---------|
| **加速检索** | 1-5 分钟 | 最高 | 需要快速访问的归档数据 |
| **标准检索** | 3-5 小时 | 适中 | 常规归档数据 |
| **批量检索** | 5-12 小时 | 最低 | 大规模数据批量导出 |
| **深度归档检索** | 12-48 小时 | 最低 | 极少访问的冷数据 |

### 检索注意事项

- 检索操作是**异步**的，需要等待完成
- 检索完成后，数据会以 **TAR 格式**提供
- 每个档案的检索次数有免费额度
- 超出免费额度后按次计费

## 数据保护与安全性

### 持久性

- S3 Glacier 提供 **99.999999999% (11个9)** 的数据持久性
- 数据在多个可用区内冗余存储
- 与 S3 Standard 具有相同的数据保护级别

### 安全性特性

| 特性 | 描述 |
|------|------|
| **加密** | 支持 SSE-S3、SSE-KMS 加密（与 [[S3 - Object Storage]] 一致） |
| **访问控制** | 通过 IAM 策略和 Vault Policy 控制访问 |
| **Vault Lock** | 一次性写入策略，防止档案被修改或删除 |
| **合规性** | 支持 HIPAA、PCI DSS、GDPR 等合规要求 |

### 数据锁定 (Locking)

- **Vault Lock** 策略可以锁定保险库，确保档案的完整性
- 一旦锁定，策略不可更改（符合 WORM - Write Once Read Many）
- 适用于：合规要求、法律保留、审计日志

## 生命周期管理

### S3 生命周期策略

- Glacier 作为 [[S3 - Object Storage]] 生命周期规则的目的地
- 自动将旧对象从 Standard 转移到 Glacier
- 可配置规则：
  - 基于时间的转移（如 30 天后转移到 Glacier）
  - 基于前缀或标签的转移
  - 过期后自动删除

### 典型生命周期配置

```
创建 → S3 Standard (热数据)
        ↓ (30天后)
     S3 IA (冷数据)
        ↓ (90天后)
     S3 Glacier (归档)
        ↓ (180天后)
     S3 Glacier Deep Archive (长期归档)
```

## 使用场景

### 典型应用场景

1. **合规和法规要求**：需要长期保留的金融、医疗记录
2. **灾难恢复**：备份和灾难恢复的长期存储
3. **视频归档**：监控录像、媒体内容存档
4. **数据仓库**：历史数据分析的冷数据存储
5. **审计日志**：满足合规要求的审计日志长期保留

### 不适合的场景

- ❌ 需要频繁访问的数据
- ❌ 实时数据分析
- ❌ 需要低延迟响应的应用

## 最佳实践

1. **选择合适的存储类别**：根据访问频率选择 Glacier Flexible Retrieval 或 Deep Archive
2. **配置生命周期策略**：自动将数据从热存储转移到 Glacier 降低成本
3. **使用 Vault Lock**：对需要合规保护的档案使用锁定策略
4. **启用加密**：使用 KMS 加密保护归档数据
5. **选择适当的检索类型**：平衡检索速度和成本需求
6. **监控存储成本**：设置 CloudWatch 告警监控 Glacier 存储使用量

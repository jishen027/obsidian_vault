## 核心定义
- 流量分发: 它充当流量的“指挥官”，将请求引导至健康的后端服务器
- 健康检查: 它会定期测试后端资源的运行状况。如果某个实例发生故障，负载均衡器会自动停止向其发送流量，直到它恢复正常
- [[Auto Scaling]]:与 Auto Scaling 紧密集成，当新实例启动以应对高需求时，负载均衡器会自动将流量分发给这些新实例

## 类型
### ALB 
运行在 OSI 模型的第 7 层（应用层），**专门处理 HTTP 和 HTTPS 流量**。它支持基于内容（如路径或主机名）的高级路由规则
- **高级路由功能：** 与传统负载均衡器不同，ALB 可以根据请求的内容进行智能分发。它支持**基于路径的路由**（例如将 `/api` 的请求发往一组服务器，将 `/images` 发往另一组）以及**基于主机名**的路由
- **粘性会话 (Sticky Sessions)：** ALB 支持通过 **Cookie** 实现会话亲和性。这确保了在会话期间，来自特定用户的所有连续请求都会被路由到同一台后端服务器上，非常适合需要保留本地缓存或状态的应用
- **健康检查：** ALB 能够针对目标组执行深入的健康检查。它可以检查特定的 HTTP 响应代码（通常是 200 到 499 之间），如果某个实例未通过检查，ALB 会自动停止向其发送流量，直到它恢复正常
- **深度集成：** 它与 **Auto Scaling** 组完美配合，每当 Auto Scaling 启动新实例时，都会自动将其注册到 ALB 的目标组中。同时，它也原生支持**容器化应用**（如 Amazon ECS），能够灵活地管理动态端口映射
- **安全性：** ALB 支持 SSL/TLS 卸载，您可以在 ALB 上集中管理加密证书（通过 AWS Certificate Manager），从而减轻后端服务器的计算负担
**核心应用场景：** 如果您正在构建**微服务架构**或需要根据 URL 内容灵活分发流量的复杂 Web 应用，ALB 是最佳选择
### NLB
运行在第 4 层（传输层），**处理 TCP、UDP 和 TLS 协议**。它专为极高性能、低延迟和应对突发流量（每秒数百万个请求）而设计

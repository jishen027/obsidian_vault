Lambda function 代表无服务架构的核心
1. 无服务架构的定义： 高度抽象化，只需要部署代码
2. 运行机制和限制
	1. 事件驱动： 代码被触发时运行
	2. 执行时长： 最长可15分钟运行
	3. 自动管理： 操作完成，资源关闭
3. 可靠性和可用性： 原生高可用
4. 性能效率和成本优化
	1. 极小的空间占用
	2. 按需付费
5. 计算服务典型用例
	1. 从后端数据库检索数据
	2. 解析实时数据流
	3. 执行轻量级的事务处理
	4. 自动化运维任务
	5. 如果SSA 考试中提到自动扩展，无需维护服务器，针对间歇性任务进行成本优化通常是Lambda Function

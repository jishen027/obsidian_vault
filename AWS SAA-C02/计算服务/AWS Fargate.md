**AWS Fargate** 是一种专为容器设计的**无服务器（Serverless）**计算引擎，目前支持 Amazon [[ECS]] 和 Amazon EKS

## 核心定义：无服务器容器化
在传统的容器运行模式中，你需要自己管理底层的 [[EC2]] 实例（即服务器）。而 Fargate 的核心价值在于：它允许你直接运行容器，而**无需管理任何服务器**。你只需要打包你的应用镜像，定义 CPU 和内存的需求，AWS 就会自动处理底层的计算基础架构

## 与 ECS 结合的工作逻辑
在 Amazon [[ECS]] 中，Fargate 是两种运行模式之一（另一种是 EC2 模式）
- **任务定义 (Task Definition)：** 你在 JSON 模板中指定该任务需要使用 Fargate 运行
- **资源配额：** 你需要显式定义容器所需的 CPU（如 0.25 vCPU）和内存（如 0.5 GB）
- **自动缩放：** 配合 Auto Scaling，Fargate 可以根据实时需求秒级启动新容器，响应速度极快

## Fargate 的显著优势
- **管理负担极低：** 你不再需要执行打补丁、扩容集群或管理实例 OS 等繁琐的运维任务
- **资源利用率高：** Fargate 采用“装箱”技术，在单台受管实例上紧密部署多个负载，确保存储和计算资源被充分利用
- **安全性增强：** 每个 Fargate 任务都运行在独立的内核隔离计算环境中，不同任务之间不共享内存或 CPU，提供了极高的安全边界
- **按需付费：** 你只需为你定义的容器资源和运行时间付费，没有闲置实例带来的浪费

## 成本优化建议
对于长期运行的 Fargate 负载，AWS 提供了 **Compute Savings Plans**。如果你承诺使用 1-3 年，最高可获得比按需价格低 **66%** 的折扣

## 什么时候该选 Fargate 而不是 EC2？
- **选 Fargate：** 追求极致简便、想减少运维开销、或是处理突发性不规律负载时
- **选 [[EC2]] 模式：** 如果你需要完全控制底层实例（如特定内核调整）、需要使用 GPU 硬件、或者有极大规模且稳定的负载需要通过预留实例实现极致压价时

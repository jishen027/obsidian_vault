# AWS Device Farm

> **AWS Device Farm** 是一项完全托管的服务，用于在 AWS 云中的真实物理设备上测试 Android、iOS 和 Web 应用程序。它是移动应用开发和测试的核心服务。
>
> 相关文档：[[Lambda]] | [[Elastic Beanstalk]]

---

## 核心概念

### Device Farm 定位

| 服务 | 类型 | 场景 | 特点 |
|------|------|------|------|
| **AWS Device Farm** | 移动测试 | Android/iOS 应用测试 | 真实设备、并行测试 |
| **AppStream 2.0** | 应用流式传输 | 桌面应用远程访问 | 浏览器访问桌面应用 |
| **AWS Lambda** | Serverless 计算 | 事件驱动函数执行 | 无需管理服务器 |

### Device Farm vs AppStream 2.0 对比（考试高频考点）

| 特性 | AWS Device Farm | AppStream 2.0 |
|------|----------------|---------------|
| **主要用途** | 移动应用测试 | 桌面应用流式传输 |
| **支持平台** | Android、iOS、Web | Windows、Linux 桌面应用 |
| **设备类型** | 真实物理设备 | 虚拟桌面 |
| **测试功能** | 并行测试、日志、性能数据 | 远程访问、协作 |
| **适用场景** | 兼容性测试、性能测试 | 远程办公、专业软件 |

## 核心功能

### 测试类型

| 测试类型 | 说明 | 适用场景 |
|---------|------|---------|
| **全屏测试** | 在真实设备上运行 UI 自动化测试 | Appium、XCTest、Cucumber |
| **自定义测试** | 运行自定义脚本和测试框架 | 特定测试需求 |
| **实时测试** | 通过浏览器远程控制真实设备 | 手动探索性测试 |
| **视频录制** | 自动录制测试过程 | 问题重现和调试 |

### 设备支持

| 平台 | 支持设备 | 测试框架 |
|------|---------|---------|
| **Android** | Samsung、Google、OnePlus 等主流设备 | Appium、UIAutomator、Cucumber |
| **iOS** | iPhone、iPad 最新型号 | XCTest、XCUITest、Appium |
| **Web** | 多种浏览器和屏幕尺寸 | Selenium、WebDriver |

### 测试监控功能

| 功能 | 说明 |
|------|------|
| **日志记录** | 详细的测试日志和错误信息 |
| **性能数据** | CPU、内存、网络、电池等指标 |
| **截图和视频** | 自动捕获测试过程和结果 |
| **网络 throttling** | 模拟不同网络条件（3G、4G、WiFi） |
| **地理位置模拟** | 模拟不同地理位置 |
| **方向旋转** | 测试不同屏幕方向 |

## 集成与自动化

### CI/CD 集成

```
代码提交 → CodeCommit → CodeBuild → CodePipeline → Device Farm 测试 → 部署
```

### 支持的集成服务

| 服务 | 集成方式 | 说明 |
|------|---------|------|
| **AWS CodePipeline** | 测试阶段插件 | 自动在发布前执行测试 |
| **AWS CodeBuild** | 构建步骤 | 构建测试包并上传 |
| **Jenkins** | 插件 | 外部 CI/CD 集成 |
| **命令行工具** | AWS CLI | 自定义自动化流程 |

### CodePipeline 集成示例

```
Source（CodeCommit）→ Build（CodeBuild）→ Test（Device Farm）→ Deploy（Elastic Beanstalk）
```

- **测试阶段**：Device Farm 作为测试阶段插件
- **自动执行**：每次代码提交后自动运行测试
- **质量门禁**：测试失败阻止部署到生产环境

## 定价模型

### 计费项

| 计费项 | 说明 |
|--------|------|
| **设备分钟数** | 按设备使用时间计费 |
| **数据存储** | 测试视频和日志存储 |
| **网络传输** | 测试包上传和结果下载 |

### 成本优化建议

1. **并行测试**：同时使用多个设备减少总测试时间
2. **选择合适的设备**：只测试目标用户使用的设备
3. **使用免费套餐**：新用户可获得免费试用额度
4. **优化测试包**：减小测试包大小降低传输时间

## 考试重点总结

### SAA-C02 高频考点

1. **Device Farm 用途**：Android/iOS 应用测试，真实物理设备
2. **与 AppStream 2.0 区别**：Device Farm 用于测试，AppStream 用于应用流式传输
3. **关键词识别**："Android 和 iOS"、"真实设备"、"日志和性能数据" → Device Farm
4. **CI/CD 集成**：CodePipeline 测试阶段插件
5. **测试功能**：并行测试、日志、性能数据、截图/视频
6. **支持平台**：Android、iOS、Web
7. **实时测试**：通过浏览器远程控制真实设备
8. **网络模拟**：模拟 3G、4G、WiFi 等网络条件

### 场景题解题思路

```
场景分析 → 选择测试服务
├── "Android 和 iOS 测试" → AWS Device Farm
├── "真实设备" → AWS Device Farm
├── "日志和性能数据" → AWS Device Farm
├── "浏览器访问桌面应用" → AppStream 2.0
└── "远程办公" → AppStream 2.0
```

### 常见考试场景分析

**场景：公司需要测试其移动应用在多种 Android 和 iOS 设备上的兼容性和性能，需要重现用户报告的问题并确保应用质量，最具成本效益的方案？**

| 选项 | 是否正确 | 原因 |
|------|---------|------|
| **AWS Device Farm** | ✅ 正确 | 真实设备测试，支持并行测试，提供详细日志和性能数据 |
| AppStream 2.0 | ❌ 错误 | 用于桌面应用流式传输，不是移动应用测试服务 |
| Amazon EC2 | ❌ 错误 | 虚拟机，无法测试移动设备兼容性 |
| AWS Elastic Beanstalk | ❌ 错误 | 应用部署服务，不是测试服务 |

**关键总结：**
- **"Android 和 iOS"** → Device Farm
- **"真实设备"** → Device Farm
- **"日志和性能数据"** → Device Farm
- **"浏览器访问桌面应用"** → AppStream 2.0

## 最佳实践

1. **选择合适的设备**：根据目标用户群体选择设备
2. **并行测试**：同时使用多个设备减少测试时间
3. **集成 CI/CD**：使用 CodePipeline 自动化测试流程
4. **模拟真实环境**：使用网络 throttling 模拟不同网络条件

AWS KMS 负责集中管理加密密钥的核心托管服务
## 核心功能和密钥管理
- 生命周期控制： KMS 允许用户集中管理加密密钥的创建，跟踪，轮换，禁用和删除
- 密钥类型区分
	- AWS 托管密钥： 由AWS自动创建并且管理，通常每年轮换一次，用户无法控制
	- 客户管理密钥 - CMK ： 用户拥有完全控制权，自定义轮换周期和手动禁用或者撤销密钥
- 授权机制： KMS 必须通过密钥策略 - Key policies 进行授权, 该策略划分了密钥管理员和密钥用户，进行严格的最小特权原则

## 服务集成和数据保护
- 落盘数据加密 - Encryption at Rest : KMS广泛集成AWS 核心服务， 为S3[[S3 - Object Storage]] - SSE-KMS, [[块储存 - EBS]] 和 [[文件储存 - AWS EFS]]， 以及 [[关系型数据库 - RDS]] 和 [[DynamoDB - NoSQL]]提供透明的加密保护
- 开发工具安全： 即使时CodeCommit存储库，也会使用KMS 管理的密钥对静态代码进行自动加密
- 合规和审计：KMS和AWS CloudTrail深度集成，每一次密钥的使用行为都会被记录在案，为企业提供完整的审计追踪

## 在安全性工具链中的定位
- 与CloudHSM的区别： KMS时通用的托管加密服务， 而CloudHSM专为需要专用第三方验证硬件模块的高合规工作负债设计
- 与Secrets Manager的协作： KMS专注于管理加密密钥，而Secrets Manager侧重于管理凭证， 如数据库密码， 但是在存储这些凭证，Secrets Manager 会调用KMS进行底层加密
